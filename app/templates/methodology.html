Assessment Methodology

Aegis WebSec is a detection-only security assessment platform for authorized targets. It combines scoped crawling, modular checks, CVSS-style scoring, evidence exports, and remediation planning.

Authorization First Same-Origin Scope CVSS-Style Scoring Evidence Package Retest Workflow

Scope Control

Targets are normalized, validated, and scanned within controlled page/depth limits. Crawling stays same-origin to avoid accidental scope expansion.

Detection Modules

Modules review headers, TLS, cookies, CSP, CORS, security.txt, robots exposure, forms, directories, SQLi, XSS, redirects, and mixed content.

Reporting Model

Findings are enriched with CVSS-style severity, priority, SLA, confidence, exploitability, business impact, data sensitivity, and verification status.

Scan Pipeline

Validate authorization-oriented target input and optional policy blocklist.
Crawl same-origin pages within selected depth and page limits.
Run passive and active modules based on the selected profile.
Normalize findings into risk, control, effort, and workflow metadata.
Persist history and generate PDF, JSON, CSV, and portfolio artifacts.

Competition Differentiators

  • Dashboard, history, result detail, PDF, JSON, CSV, portfolio API, and readiness API are all implemented.
  • Executive and technical audiences get different views of the same evidence.
  • Remediation workflow and retest planning make the project operational, not just visual.
  • Ethical-use framing and scope controls are visible in the product and documentation.
